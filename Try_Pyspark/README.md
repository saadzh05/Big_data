# PySpark Framework – Dépôt de Tests des Stagiaires

## Description

Ce dépôt est **exclusivement dédié aux tests des stagiaires**.  
Il a pour objectif de permettre aux stagiaires de **pousser (push) et mettre à jour leurs travaux** liés au **framework PySpark**, dans le cadre des évaluations pédagogiques.

Chaque stagiaire doit utiliser ce dépôt pour :
- Déposer ses scripts PySpark
- Tester les fonctionnalités du framework fourni
- Valider ses implémentations à travers des cas pratiques
- Faciliter la correction et le suivi des travaux par les formateurs

---

## Objectifs pédagogiques

- Évaluer la maîtrise de **PySpark**
- Vérifier la compréhension des concepts clés :
  - RDD, DataFrame, Spark SQL
  - Transformations et Actions
  - Optimisation et performance
- Tester la capacité à travailler avec un **framework structuré**
- Encourager les bonnes pratiques de développement (code clair, structuré et documenté)

---

## Instructions pour les stagiaires

1. Forker ou cloner le dépôt
2. Créer un dossier à votre **nom_prenom** dans `stagiaires/`
3. Implémenter vos tests et scripts PySpark
4. Vérifier l’exécution correcte du code
5. Pousser (push) vos modifications

---

## Règles importantes

-  Aucun code ne doit être modifié dans le dossier `framework/`
-  Le travail d’un autre stagiaire ne doit pas être modifié
-  Le code doit être lisible, commenté et fonctionnel
-  Tout plagiat entraînera une annulation de l’évaluation

---

## Évaluation

Les critères d’évaluation incluent :
- Correctitude des résultats
- Qualité du code
- Respect des consignes
- Organisation et clarté
- Utilisation appropriée de PySpark
